Experiment 7: Constructors and Destructors

Aim

To explain the Constructors and Destructors. In this experiment, the Student will be able to understand what are constructors and destructors in C++ programming Language with the help of an iterative simulator.

Theory

Constructor initializes an object when created, while a destructor cleans up resources when the object is destroyed.
Constructor
- Purpose: Initializes an object.
- Example: `MyClass(int value)` initializes the `num` member variable with `value`.
Destructor
- Purpose: Cleans up resources when an object is destroyed.
- Example: `~MyClass()` outputs "Destructor called" when the object goes out of scope.
Copy Constructor
- Purpose: Creates a new object as a copy of an existing object.
- Example: `MyClass(const MyClass &obj)` copies the `num` value from `obj` and prints a message.
Default Constructor vs. Copy Constructor
- Default Constructor: Automatically generated if no constructors are defined. Initializes members with default values.
- Copy Constructor: Defined explicitly to create a new object as a copy of another object, often used for deep copying.
In the provided code, the `copy constructor` is called when copying an object, and the `destructor` is invoked when the object is destroyed.
